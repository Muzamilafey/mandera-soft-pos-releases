#  Security Policy

## 🛡 Supported Versions

We actively maintain and provide security updates for the latest stable versions of Mandera Soft POS.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

> We strongly recommend always using the latest available version to ensure maximum security and performance.

---

##  Reporting a Vulnerability

We take security seriously. If you discover a vulnerability, please report it responsibly.

###  How to Report

Send an email to:

**[muzamilinc@gmail.com](mailto:muzamilinc@gmail.com)**

Include the following details:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Screenshots or proof of concept (if available)

---

## ⏱ Response Time

* Initial response: within **24–72 hours**
* Status updates: provided regularly until resolution
* Critical issues are prioritized and handled immediately

---

##  What to Expect

* You will receive confirmation once your report is reviewed
* If the issue is valid, we will work on a fix as soon as possible
* You may be contacted for additional information
* Once resolved, a patch or update will be released

---

##  Do Not

To protect users, please:

* Do NOT publicly disclose the vulnerability before it is fixed
* Do NOT attempt to access or modify user data
* Do NOT perform destructive testing

---

##  Responsible Disclosure

We appreciate responsible disclosure and will acknowledge contributors who help improve the security of our system.

---

##  Scope

This policy applies to:

* Mandera Soft POS Desktop Application
* Associated backend APIs and services

---

##  Notes

* This repository contains only the desktop application installer
* Core source code is private and maintained securely
